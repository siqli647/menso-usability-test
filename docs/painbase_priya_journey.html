Recommended Fixes

1. Reposition for evidence-backed discovery.
Add a PM/team-oriented path: "Find market pain you can defend in roadmap planning." Keep MVP generation, but do not make it the only hero promise.
2. Fix source integrity before scaling AI output.
Every card should expose source URL, excerpt, author, date, platform, and confidence rationale. Prevent placeholder IDs from appearing in production data.
3. Make Search failure explicit and recoverable.
If no results are found, show "No results" with query suggestions. If generation fails, show reason and retry state. Never leave the main pane in "Start Your Search" after submission.
4. Promote Pain Point Details as the PM workflow.
Severity, urgency, score, audience, related signals, and source proof are the decision artifact Priya wants. Make this view shareable/exportable for stakeholder review.
5. Testing caveat: exclude My Projects confusion.
The projects Priya saw were generated by previous Menso tests using the same account. This should not be counted as a PainBase usability defect; future runs should start from a clean account or cleared project history.
6. Attach validation guidance to generated MVPs.
The generated landing page is useful, but Priya needs a path to validation: who to test with, which assumptions to check, and how source evidence supports the copy.
