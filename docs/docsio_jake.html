What Docsio Solves

Time-to-docs for founders

Jake wants to paste a product/docs URL and avoid spending hours maintaining Docusaurus structure, styling, and deployment.

AI-assisted maintenance

The core promise is not only first generation, but ongoing changes through chat: colors, new pages, rewritten guides, and publishing.

Docs infrastructure removal

The buyer wants generated structure, hosted preview, GitHub sync, and publish settings without assembling a docs stack by hand.

Core Evidence

"I expected to paste my URL and see generated docs within minutes. Instead, I wasted time navigating and feeling misled."

Top Findings

Critical

AI editing limit blocks evaluation

The existing project immediately showed "Monthly limit reached (5/5)", so Jake could not test the key proof points: adding an API Reference page or rewriting Getting Started. For a $60/month decision, this removes the trial evidence he needs.

Major

No visible new-project action after login

The dashboard showed one draft project and no obvious "New Project" or URL input. That contradicts the landing-page promise of a fast URL-to-docs workflow.

Major

Marketing promise and logged-in state diverge

The homepage sells "30-second setup"; the returning-user dashboard made Jake hunt across dashboard, project editor, and settings without finding the expected starting point.

Product Recommendations

  • Add a persistent "New docs site" button and URL input on the dashboard, even when a draft already exists.
  • Show plan limits before users enter a project, and reserve one end-to-end generation/edit path for evaluation.
  • When limits are reached, offer a read-only demo or resettable sandbox so buyers can still inspect the core workflow.
  • Route returning users to the fastest next action, not only to their most recent draft.
