Priority Recommendations

1. Fix Deep Research reliability and recovery.
Add timeout, persistent cancel, retry, switch-to-Normal fallback, estimated wait time, and async completion. A stuck research job breaks Elena's core repeated-screening workflow.
2. Label generated proof as placeholders.
Testimonials, pricing, brand domains, and claims should carry visible "AI-generated placeholder" labels and a replace-before-sharing checklist.
3. Expand gap analysis into decision evidence.
Add signal frequency, competitor landscape, current workaround, addressability, and why existing solutions are insufficient. Keep the default scannable, but support deeper expansion.
4. Clarify preview, export, and hosting semantics.
Make it explicit whether View Live is a local preview, a hosted share page, or a deployable asset. If a domain is only suggested, label it as suggested and show next steps.
5. Upgrade My Build Lab for multi-project teams.
Add project status, tags, folders, owner/client fields, archive/delete, team sharing, and batch export so the workspace supports studio operations.
6. Make source evidence easier to verify inline.
Show source excerpt, platform, author, date, URL, and cluster context without forcing the user to leave PainBase for every trust check.
